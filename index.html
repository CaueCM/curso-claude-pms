Bloco 01 · 09:00–09:45

Fundamentos: Chat vs. Cowork vs. Code

"Você não está usando o Claude. Está conversando com ele. Tem uma diferença enorme."

Objetivo: Nivelar o grupo e construir o modelo mental correto. Todo mundo sai sabendo quando usar Chat, Cowork e Code.

Parte expositiva — O mapa do território 15 min

Claude Chat

Você pergunta, ele responde. Bom para brainstorm, análise rápida, reescrita. Não tem memória entre sessões, não mexe em arquivos, não conecta com nada.

= ligar para um colega e pedir opinião

Claude Cowork

Você delega, ele executa. Lê e cria arquivos, encadeia tarefas, conecta com Gmail/Slack/Drive, agenda rotinas.

= delegar tarefa a um assistente com acesso à sua mesa

Claude Code

Você descreve, ele constrói. Terminal que entende português. Cria protótipos, ferramentas, sites.

= sentar com um dev e ir construindo junto

Onde cada interface funciona

InterfaceDesktopWeb (claude.ai)Mobile
Chat✅ completo✅ completo✅ completo
Cowork✅ completo❌❌
Code✅ via terminal❌❌

Regra prática: trabalho real = Desktop. Mobile = satélite.

Exercícios

01 Setup e primeiro contato com a ArenaCash 15 min
  1. Criar pasta ArenaCash-PM no computador
  2. Copiar todos os arquivos do kit para dentro da pasta
  3. Abrir Claude Cowork e conectar à pasta
  4. Pedir: "Me diz o que tem nessa pasta e o que você consegue fazer com ela"

  6. Pedir: "Lê o memo estratégico e o organograma e me faz um resumo executivo da ArenaCash neste quarter"
  7. Observar: cruzou dois documentos e entregou contexto — isso vai ser útil o dia inteiro
02 Prompting vs. delegação: qualidade e custo 15 min

Três rodadas com o mesmo objetivo ("entender os dados de churn da ArenaCash"):

Rodada 1 · Chat

Prompt vago: "me fala sobre o churn da ArenaCash"

Rodada 2 · Chat

Prompt estruturado: "analise o churn do Q1, segmente por plano, identifique os 3 maiores motivos"

Rodada 3 · Cowork

Delegação com contexto: "na pasta tem dados-quarter-q1.csv. Analisa o churn, cria planilha segmentada por plano e relatório com os 3 maiores motivos e recomendações."

Comparar: qualidade lado a lado + consumo de tokens (Settings → Usage). Prompt vago obriga a re-promptar 3-4 vezes. Delegar bem é mais barato.

📄 01-fundamentos/comparativo-prompts.md

Sessão · 5 min

Como pensar em tokens

"Token é como minuto de reunião: o que se mede, se gerencia."

1

Prompt bom de primeira = economia

Um prompt vago vira 4-5 idas e vindas. Cada turno carrega toda a conversa.

2

Contexto no lugar certo

Instruções no Projeto > repetir contexto. Referenciar arquivos > colar conteúdo. Output em arquivo > "me mostra na tela".

3

Conversas curtas

Terminou a tarefa, abra nova. Edite em vez de regerar — se 90% tá bom, ajuste o parágrafo.

4

Modelo certo pra tarefa

Opus = caro, problemas complexos. Sonnet = dia a dia. Haiku = tarefas leves. Skills entregam bem de primeira.

Bloco 02 · 09:50–10:40

Setup do PM na ArenaCash

"O segredo não é o prompt. É o contexto que você dá antes do prompt."

Objetivo: Montar o ambiente completo de trabalho: projeto com memória, instruções, organograma como referência, tom de voz e primeira skill.

A arquitetura do Cowork 10 min

Projetos

Workspace dedicado: pasta, instruções, memória, tasks. Memória é isolada por projeto.

Sistema de memória

Instruções (você escreve) + Auto memory (Claude salva padrões em memory.md).

Taxonomia

Projetos → Tarefas → Skills → Agents → Plugins.

Projeto
├── Instruções (.md) ← você escreve, Claude lê toda sessão
├── Memória (.md)    ← Claude escreve, persiste entre sessões
├── Tarefas
│   ├── Avulsas
│   └── Agendadas (scheduled tasks)
├── Skills (nativas, customizadas, da comunidade)
├── Connectors (Slack, Gmail, Drive...)
└── Plugins (bundles de skills + connectors)

Bloco 03 · 10:55–11:40

MCP, dados e priorização de iniciativas

"O Claude sozinho é bom. Plugado nas suas ferramentas, é transformador."

Objetivo: Entender MCP, conectar ferramentas, usar os dados da ArenaCash para gerar e priorizar iniciativas com o thinking partner.

O que é MCP 10 min

MCP = Model Context Protocol. O Claude é um cérebro trancado numa sala. MCP é o corredor que liga a sala às suas ferramentas. Protocolo aberto (Anthropic).

Você pede: "Busca nos meus emails o que a Maria mandou"
    ↓
Claude entende o que você quer
    ↓
Claude chama o Connector do Gmail via MCP
    ↓
Connector autentica (OAuth)
    ↓
Connector busca e devolve
    ↓
Claude analisa e entrega o resumo

O que fazem

Ler dados · criar rascunhos · buscar · filtrar

O que NÃO fazem sem confirmação

Enviar emails · postar · deletar · mudar permissões

Bloco 05 · 13:45–14:30

Do Cowork ao Code: transição e setup

"A internet inteira fala de Code. Mas entenda quando vale e quando é matar mosca com canhão."

Objetivo: Entender quando sair do Cowork pro Code, instalar, e mostrar que os arquivos da manhã continuam funcionando.

Quando Code faz sentido 15 min

1. Prototipação funcional

Code cria coisas que rodam: telas, dashboards, sites interativos.

2. Conexão com Git

Lê repositórios, cria branches, commita, abre PRs.

3. Análise de código

Explica em português: "esse endpoint faz o quê?"

4. SQL e queries

Conecta em banco, executa queries, gera gráficos.

5. MCP servers customizados

Qualquer API vira MCP server: Metabase, Amplitude, banco interno.

6. Ferramentas standalone

Calculadoras, dashboards, scripts que rodam sozinhos.

7. Hooks e validação

Scripts antes/depois de ações. "Antes de commitar, verifica seções do PRD."

Matriz de decisão

CenárioCoworkCode
Documentos (Word, Excel, PPT)✅ melhorpossível
Email / Slack✅ connectorsvia MCP
Tasks agendadas✅ nativopossível
Protótipos funcionais❌✅ melhor
Git❌✅ melhor
SQL e dados pesados❌✅ melhor
MCP customizados❌✅ melhor
Ferramentas standalone❌✅ melhor
Hooks e automação❌✅ melhor

Regra de bolso: output é arquivo → Cowork. Output é ferramenta ou envolve código → Code.
