Você foi contratado como PM de produto.

A ArenaCash é uma fintech criada pela Product Arena. Ela não existe de verdade — mas os problemas que você vai resolver aqui são os mesmos que surgem toda semana no trabalho real de um PM.

Antes do curso, você recebeu o ArenaCash Kit: memo estratégico do CEO, transcrições das conversas com CFO, CMO e CTO, dados de produto e financeiros do quarter anterior, tickets de atendimento, design system e template oficial de apresentação.

Fundamentos: Chat vs. Cowork vs. Code

"Você não está usando o Claude. Está conversando com ele. Tem uma diferença enorme."

Objetivo: Nivelar o grupo e construir o modelo mental correto. Todo mundo sai sabendo quando usar Chat, Cowork e Code.

Parte expositiva — O mapa do território 15 min

Claude Chat

Você pergunta, ele responde. Brainstorm, análise rápida, reescrita. Não tem memória entre sessões, não mexe em arquivos, não conecta com nada.

= ligar para um colega e pedir opinião

Claude Cowork

Você delega, ele executa. Lê e cria arquivos, encadeia tarefas, conecta com Gmail/Slack/Drive, agenda rotinas.

= delegar tarefa a um assistente com acesso à sua mesa

Claude Code

Você descreve, ele constrói. Terminal que entende português. Cria protótipos, ferramentas, sites.

= sentar com um dev e ir construindo junto

Onde cada interface funciona

InterfaceDesktopWeb (claude.ai)Mobile
Chat✅ completo✅ completo✅ completo
Cowork✅ completo❌❌
Code✅ via terminal❌❌

Regra prática: trabalho real = Desktop. Mobile = satélite.

Como pensar em tokens

"Token é como minuto de reunião: o que se mede, se gerencia."

1

Prompt bom de primeira = economia

Um prompt vago vira 4-5 idas e vindas. Cada turno carrega toda a conversa.

2

Contexto no lugar certo

Instruções no Projeto > repetir contexto. Referenciar arquivos > colar conteúdo. Output em arquivo > "me mostra na tela".

3

Conversas curtas

Terminou a tarefa, abra nova. Edite em vez de regerar — se 90% tá bom, ajuste o parágrafo.

4

Modelo certo pra tarefa

Opus = caro, problemas complexos. Sonnet = dia a dia. Haiku = tarefas leves. Skills entregam bem de primeira.

Setup do PM na ArenaCash

"O segredo não é o prompt. É o contexto que você dá antes do prompt."

Objetivo: Montar o ambiente completo de trabalho: projeto com memória, instruções personalizadas com o organograma como referência, tom de voz calibrado e integração com Gmail.

A arquitetura do Cowork 10 min

Projetos

Workspace dedicado: pasta, instruções, memória, tasks. Memória é isolada por projeto.

Sistema de memória

Instruções (você escreve) + Auto memory (Claude salva padrões em memory.md).

Taxonomia

Projetos → Tarefas → Skills → Agents → Plugins.

Projeto
├── Instruções (.md) ← você escreve, Claude lê toda sessão
├── Memória (.md)    ← Claude escreve, persiste entre sessões
├── Tarefas
│   ├── Avulsas
│   └── Agendadas (scheduled tasks)
├── Skills (nativas, customizadas, da comunidade)
├── Connectors (Slack, Gmail, Drive...)
└── Plugins (bundles de skills + connectors)

O que é MCP 10 min

MCP = Model Context Protocol. O Claude é um cérebro trancado numa sala. MCP é o corredor que liga a sala às suas ferramentas. Protocolo aberto da Anthropic.

Você pede: "Busca nos meus emails o que a Maria mandou"
    ↓
Claude entende o que você quer
    ↓
Claude chama o Connector do Gmail via MCP
    ↓
Connector autentica (OAuth)
    ↓
Connector busca e devolve
    ↓
Claude analisa e entrega o resumo

O que fazem

Ler dados · criar rascunhos · buscar · filtrar

O que NÃO fazem sem confirmação

Enviar emails · postar · deletar · mudar permissões

08Criando o Thinking Partner com C-Levels15 min

No Bloco 3 os alunos usaram o thinking partner com uma skill pré-instalada. Agora vão entender como ela funciona por dentro — e criar a própria.

Uma Skill é um arquivo markdown com instruções detalhadas que ensinam o Claude a fazer uma tarefa específica. A analogia: "Você manda briefing de 2 linhas pra um designer e espera trabalho incrível? Não. Skill é o briefing completo, escrito uma vez e reutilizado pra sempre."

P R O M P T   P A R A   O   C L A U D E

Cria uma skill chamada thinking-partner:

  • Trigger: quando eu pedir para discutir, avaliar ou validar uma decisão
  • O que faz: lê organograma + transcricao-cfo.md + transcricao-cmo.md + transcricao-cto.md. Cria 3 personas com perfil, preocupações e estilo. Apresenta o ponto de vista de cada um, destacando concordância e divergência.
  • Output: análise com 3 perspectivas + síntese de consenso + pontos de atenção
  • Regras: sempre baseado nas transcrições, nunca inventar posições

💡 Comparar com a skill pré-instalada do Bloco 3: são idênticas ou têm diferenças? Por quê?

Do Cowork ao Code: transição e setup

"A internet inteira fala de Code. Mas entenda quando vale e quando é matar mosca com canhão."

Objetivo: Entender quando sair do Cowork pro Code, instalar, e mostrar que os arquivos da manhã continuam funcionando. Matar o FOMO.

Quando Code faz sentido 15 min

1. Prototipação funcional

Code cria coisas que rodam: telas, dashboards, sites interativos.

2. Conexão com Git

Lê repositórios, cria branches, commita, abre PRs.

3. Análise de código

Explica em português: "esse endpoint faz o quê?"

4. SQL e queries

Conecta em banco, executa queries, gera gráficos.

5. MCP servers customizados

Qualquer API vira MCP server: Metabase, Amplitude, banco interno.

6. Ferramentas standalone

Calculadoras, dashboards, scripts que rodam sozinhos.

7. Hooks e validação

Scripts antes/depois de ações. "Antes de commitar, verifica seções do PRD."

Matriz de decisão

CenárioCoworkCode
Documentos (Word, Excel, PPT)✅ melhorpossível
Email / Slack✅ connectorsvia MCP
Tasks agendadas✅ nativopossível
Protótipos funcionais❌✅ melhor
Git❌✅ melhor
SQL e dados pesados❌✅ melhor
MCP customizados❌✅ melhor
Ferramentas standalone❌✅ melhor
Hooks e automação❌✅ melhor

Regra de bolso: output é arquivo → Cowork. Output é ferramenta ou envolve código → Code.

Pré-requisitos

Faça isso antes do dia do curso. Cada item abaixo tem um motivo — sem ele, alguma parte do dia não funciona. Separe cerca de 30 minutos pra concluir tudo.

1
Conta Claude paga Pro, Max ou Team · sem isso o curso não começa
Obrigatório

O Claude gratuito tem limite de mensagens e não acessa o Cowork nem os plugins. O curso inteiro roda no Claude pago.

Qual plano escolher

  • Pro — funciona pro curso. É o mínimo necessário.
  • Max — recomendado se você quiser aproveitar sem se preocupar com limites. Durante um dia intenso como o curso, o Pro pode atingir o limite de mensagens. O Max tem limites muito mais altos — vai te deixar mais tranquilo pra experimentar sem segurar o ritmo. Veja a diferença entre os planos →
  • Team — se sua empresa já tem conta Team, peça ao administrador pra te adicionar. Equivalente ao Pro em limites, mas com gerenciamento centralizado.

Se ainda não tem conta: claude.ai/upgrade pra assinar.

2
Claude Desktop App interface principal do curso · só existe no desktop
Obrigatório

O Cowork — que usamos toda a manhã — só existe no Desktop. No navegador e no celular ele não aparece.

  1. Baixe o instalador em claude.ai/download (Mac ou Windows)
  2. Abra o arquivo baixado e siga a instalação normalmente
  3. Faça login com a mesma conta Claude paga do passo anterior
  4. Verifique que aparece a opção Cowork no menu lateral — se aparecer, tá tudo certo
3
Claude Code usado na parte da tarde · blocos 5 a 7
Obrigatório

Claude Code é uma ferramenta que você instala no terminal e que permite criar protótipos, sites e scripts em conversa com o Claude. É a ferramenta da parte da tarde.

Pra instalar, você vai precisar de duas coisas na ordem abaixo:

3a · Node.js (versão 18 ou mais recente)

Node.js é o motor que roda o Claude Code. Você provavelmente já tem — verifique antes de instalar.

  1. Abra o Terminal (Mac: Cmd + Espaço → "Terminal") ou PowerShell (Windows: tecla Windows → "PowerShell")
  2. Digite node --version e pressione Enter
  3. Se aparecer algo como v20.x.x ou v22.x.x, você já tem. Pule pro 3b.
  4. Se não aparecer nada: nodejs.org/en/download e baixe a versão LTS pro seu SO. Instalador padrão.

3b · Claude Code

Com o Node instalado, no mesmo terminal cole o comando abaixo e pressione Enter:

npm install -g @anthropic-ai/claude-code

Aguarde (pode demorar 1-2 minutos). Depois verifique:

claude --version

Se aparecer o número da versão, tá instalado. Documentação: docs.anthropic.com/pt/docs/claude-code.

4
Git o Claude Code depende dele por baixo dos panos
Obrigatório

Git é um sistema de controle de versão. O Claude Code usa por baixo dos panos pra gerenciar arquivos e mudanças — sem ele, o Code pode dar erro na inicialização.

  • Mac: Git já vem instalado na maioria dos casos. Verifique com git --version no Terminal. Se não tiver, o próprio Mac vai oferecer instalar automaticamente — aceite.
  • Windows: Verifique com git --version no PowerShell. Se não tiver, baixe em git-scm.com/download/win (opções padrão, não precisa mexer em nada).
5
Kit ArenaCash baixado e descompactado · é o material do curso
Obrigatório

Você vai receber o kit por email ou link antes do curso. Ele contém todos os documentos da empresa fictícia que vamos usar no dia.

  • Baixe o arquivo .zip
  • Descompacte em algum lugar fácil de achar (ex: Desktop ou Documentos)
  • Renomeie a pasta pra ArenaCash-PM
  • Verifique que consegue ver arquivos .md, .csv, .xlsx e .pptx dentro dela
6
Conta Google / Gmail usado no Bloco 2 pra extrair seu tom de voz
Recomendado

No Bloco 2 vamos conectar o Gmail ao Cowork pra extrair o seu tom de voz de escrita a partir dos seus e-mails. Funciona com qualquer conta Google.

  • Se você tem Gmail, não precisa fazer nada — só lembrar o login
  • Se preferir não usar a conta pessoal, crie uma conta separada só pro curso em accounts.google.com
  • Se não quiser usar Gmail de jeito nenhum, tudo bem — o instrutor mostra um caminho alternativo pra criar o tom de voz manualmente
