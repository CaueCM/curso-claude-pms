Você foi contratado como PM da ArenaCash.

A ArenaCash é uma fintech criada pela Product Arena. Ela não existe de verdade — mas os problemas que você vai resolver aqui são os mesmos que surgem toda semana no trabalho real de um PM.

Antes do curso, você recebeu o ArenaCash Kit: memo estratégico do CEO, transcrições das conversas com CFO, CMO e CTO, dados de produto e financeiros do quarter anterior, tickets de atendimento, design system e template oficial de apresentação.

São os mesmos materiais que um PM receberia no primeiro dia numa empresa real. Conta Claude paga Pro, Max ou Team · sem isso o curso não começa
Obrigatório

O Claude gratuito tem limite de mensagens e não acessa o Cowork nem os plugins. O curso inteiro roda no Claude pago.

Qual plano escolher

  • Pro — funciona pro curso. É o mínimo necessário.
  • Max — recomendado se você quiser aproveitar sem se preocupar com limites. Durante um dia intenso como o curso, o Pro pode atingir o limite de mensagens. O Max tem limites muito mais altos — vai te deixar mais tranquilo pra experimentar sem segurar o ritmo. Veja a diferença entre os planos →
  • Team — se sua empresa já tem conta Team, peça ao administrador pra te adicionar. Equivalente ao Pro em limites, mas com gerenciamento centralizado.

Se ainda não tem conta: claude.ai/upgrade pra assinar.

3
Claude Code usado na parte da tarde · blocos 5 a 7
Opcional

Claude Code é uma ferramenta que você instala no terminal e que permite criar protótipos, sites e scripts em conversa com o Claude. É a ferramenta da parte da tarde.

Pra instalar, você vai precisar de duas coisas na ordem abaixo:

3a · Node.js (versão 18 ou mais recente)

Node.js é o motor que roda o Claude Code. Você provavelmente já tem — verifique antes de instalar.

  1. Abra o Terminal (Mac: Cmd + Espaço → "Terminal") ou PowerShell (Windows: tecla Windows → "PowerShell")
  2. Digite node --version e pressione Enter
  3. Se aparecer algo como v20.x.x ou v22.x.x, você já tem. Pule pro 3b.
  4. Se não aparecer nada: nodejs.org/en/download e baixe a versão LTS pro seu SO. Instalador padrão.

3b · Claude Code

Com o Node instalado, no mesmo terminal cole o comando abaixo e pressione Enter:

npm install -g @anthropic-ai/claude-code

Aguarde (pode demorar 1-2 minutos). Depois verifique:

claude --version

Se aparecer o número da versão, tá instalado. Documentação: docs.anthropic.com/pt/docs/claude-code.

4

Opcional

Git é um sistema de controle de versão. O Claude Code usa por baixo dos panos pra gerenciar arquivos e mudanças — sem ele, o Code pode dar erro na inicialização.

  • Mac: Git já vem instalado na maioria dos casos. Verifique com git --version no Terminal. Se não tiver, o próprio Mac vai oferecer instalar automaticamente — aceite.
  • Windows: Verifique com git --version no PowerShell. Se não tiver, baixe em git-scm.com/download/win (opções padrão, não precisa mexer em nada).

Fundamentos: Chat vs. Cowork vs. Code

"Você não está usando o Claude. Está conversando com ele. Tem uma diferença enorme."

Objetivo: Nivelar o grupo e construir o modelo mental correto. Todo mundo sai sabendo quando usar Chat, Cowork e Code.

Parte expositiva — O mapa do território 15 min

Claude Chat

Você pergunta, ele responde. Brainstorm, análise rápida, reescrita. Não tem memória entre sessões, não mexe em arquivos, não conecta com nada.

= ligar para um colega e pedir opinião

Claude Cowork

Você delega, ele executa. Lê e cria arquivos, encadeia tarefas, conecta com Gmail/Slack/Drive, agenda rotinas.

= delegar tarefa a um assistente com acesso à sua mesa

Claude Code

Você descreve, ele constrói. Terminal que entende português. Cria protótipos, ferramentas, sites.

= sentar com um dev e ir construindo junto

Como pensar em tokens 5 min

"Token é como minuto de reunião: o que se mede, se gerencia."

No Exercício 1, você mandou um prompt claro com arquivos referenciados e recebeu um resumo executivo de primeira. Isso não é só boa prática — é economia real. Quatro princípios pra carregar o dia inteiro:

1

Prompt bom de primeira = economia

Um prompt vago vira 4-5 idas e vindas. Cada turno carrega toda a conversa acumulada.

2

Contexto no lugar certo

Instruções no Projeto > repetir contexto. Referenciar arquivos > colar conteúdo. Output em arquivo > "me mostra na tela".

3

Conversas curtas

Terminou a tarefa, abra uma nova. Edite em vez de regerar — se 90% tá bom, ajuste o parágrafo ruim.

4

Modelo certo pra tarefa

Opus = complexo. Sonnet = dia a dia. Haiku = tarefas leves. Skills = bom resultado de primeira.

Setup do PM na ArenaCash

"O segredo não é o prompt. É o contexto que você dá antes do prompt."

Objetivo: Montar o ambiente completo de trabalho: projeto com memória, instruções personalizadas com o organograma como referência, tom de voz calibrado e integração com Gmail.

A arquitetura do Cowork 10 min

1. O que é um Projeto

No Cowork, um Projeto é o seu espaço de trabalho permanente. Pense nele como uma sala dedicada pra um contexto específico: tem uma pasta conectada no seu computador, instruções que você escreve uma vez e valem pra sempre, e uma memória que persiste entre conversas.

Quando você abre uma nova task dentro de um projeto, o Claude já sabe quem você é, em que empresa trabalha e quais são as regras. Você não precisa reexplicar isso toda vez.

🔒 Regra de ouro: a memória é isolada por projeto. O que o Claude aprende no projeto da ArenaCash não vaza pra outros projetos.

2. As duas camadas de memória

📝 Instruções do Projeto

Você escreve, o Claude obedece. Regras fixas que o Claude lê no início de toda conversa. É aqui que você coloca quem é, como trabalha, quais são as convenções do time, o que nunca pode acontecer.

ex: "Nunca invente dados. Sempre salve outputs como arquivos."

🧠 Auto Memory

O Claude escreve, você pode editar. Conforme você trabalha, o Claude salva notas sobre você — suas preferências, correções, padrões. Tudo num memory.md dentro do projeto.

ex: você pode abrir, editar e apagar qualquer coisa anotada

🎬 Demo ao vivo: instrutor abre o memory.md de um projeto real e mostra o que o Claude anotou sozinho ao longo de algumas sessões de uso.

3. O ecossistema em volta do Projeto

Um Projeto não existe sozinho — ele organiza tudo mais:

Task

Uma conversa dentro do projeto. Cada task começa com o contexto do projeto carregado. Pode ser avulsa (você inicia na hora) ou agendada (roda em horários definidos).

Skill

Um pacote de instruções que ensina o Claude a produzir um tipo específico de output com qualidade consistente. Quem instala é você; quem usa é o Claude quando precisa.

Agent

Quando uma task é grande demais, o Claude quebra em partes e delega pra si mesmo em paralelo — como se fossem assistentes trabalhando ao mesmo tempo.

Connector

A ponte pra ferramentas externas (Gmail, Slack, Drive, Jira…). O Claude acessa, busca e age nessas ferramentas sem você sair do Cowork.

Plugin

Um bundle completo com skills + connectors + sub-agents empacotados. Você instala de uma vez e ganha um conjunto de capacidades prontas.

Projeto
├── Instruções (.md)   ← você escreve, Claude lê toda sessão
├── Memória (.md)      ← Claude escreve, persiste entre sessões
├── Tarefas
│   ├── Avulsas (cada conversa que você inicia)
│   └── Agendadas (rodam automaticamente)
├── Skills instaladas
│   ├── Nativas (pptx, xlsx, docx, pdf)
│   ├── Customizadas (as suas, que você vai criar hoje)
│   └── Da comunidade (instaladas do repositório)
├── Connectors (Gmail, Slack, Drive, Jira...)
└── Plugins (bundles de skills + connectors)

O que é MCP e como funciona 8 min

Nos blocos anteriores, você conectou o Gmail e o Claude buscou emails direto da sua caixa. Isso não foi mágica — foi MCP funcionando. Agora vamos entender o mecanismo.

MCP = Model Context Protocol. O Claude, por padrão, é um cérebro isolado: sabe muita coisa, mas não enxerga o que está fora da conversa. O MCP é o protocolo que abre canais entre o Claude e o mundo externo — suas ferramentas, seus dados, seus sistemas.

Você pede: "Busca no meu Drive os arquivos da ArenaCash"
    ↓
Claude entende o que você quer
    ↓
Claude chama o Connector do Google Drive via MCP
    ↓
Connector autentica com sua conta Google (OAuth — você autoriza 1 vez)
    ↓
Connector acessa o Drive, localiza os arquivos e devolve para o Claude
    ↓
Claude lê, analisa e entrega o resultado para você

O Claude nunca acessa suas ferramentas diretamente — ele sempre passa pelo Connector, que é um intermediário seguro. Você controla quais permissões cada Connector tem, e ações destrutivas (deletar, enviar mensagens, modificar permissões) sempre pedem confirmação antes de executar.

O ecossistema de Connectors hoje

  • Mais de 50 connectors nativos (Google Drive, Gmail, Slack, Notion, Jira, Linear, Figma, Salesforce, HubSpot…)
  • Qualquer empresa pode criar o próprio connector — o protocolo é aberto
  • Você já usou MCP quando conectou o Gmail no Bloco 2. O Drive é o segundo connector do dia

O que fazem

Ler e listar dados · criar rascunhos e documentos · buscar e filtrar conteúdo

O que NÃO fazem sem confirmação

Enviar emails · postar mensagens · deletar arquivos · modificar permissões

Skills, Validação e Apresentação

"Skill é o que separa o Claude genérico do Claude que trabalha do jeito que você precisa."

O que são skills e por que um PM precisa delas 10 min

O Claude, sem instrução específica, entrega resultado genérico. Com uma skill bem escrita, ele entrega resultado profissional — no padrão certo, na estrutura certa, com o nível de qualidade que você precisaria pedir dezenas de vezes pra chegar manualmente.

Pra um PM, isso importa especialmente em 2 momentos

🎯 Validação de iniciativas

O dia a dia é cheio de ideias que precisam ser testadas contra a realidade da empresa antes de avançar. Quanto mais rápido você simula esse teste — com os stakeholders certos, na perspectiva certa — menos tempo gasta defendendo a ideia errada. Uma skill de validação faz isso em minutos.

📣 Comunicação recorrente

PPTs pra diretoria, relatórios de quarter, briefings de sprint — documentos que seguem sempre o mesmo padrão. Uma skill aprende esse padrão uma vez e aplica em todo output, sem você re-explicar template, tom de voz ou nível de detalhe.

Os 3 tipos de skill

Nativas

Já vêm com o Claude e rodam automaticamente: pptx, xlsx, docx, pdf. Você já usou sem perceber.

Comunidade

Criadas e compartilhadas por outros usuários. Ponto de partida: github.com/BehiSecc/awesome-claude-skills.

Customizadas

As suas. Quando o padrão genérico não serve: template da empresa, processo específico, output só no seu contexto.

⚖️ Quando criar vs. instalar: se já existe uma skill boa, instale e adapte. Crie do zero quando: você repete a mesma tarefa 3+ vezes, o output precisa seguir padrão da empresa, ou a qualidade genérica não é suficiente.

Como uma skill funciona por dentro

É um arquivo markdown (.md) salvo em .claude/skills/nome-da-skill/SKILL.md. Contém:

  • Trigger — quando a skill deve ser acionada
  • Fluxo — o que ela deve fazer passo a passo
  • Output — qual é o resultado esperado
  • Regras — o que nunca pode ser quebrado

O Claude lê esse arquivo e segue as instruções como se fossem um briefing permanente.
